# z2m - Руководство пользователя

![z2m Icon](../static/z2m.png "Плагин z2m")

## Назначение

`z2m` подключает osysHome к Zigbee-устройствам через Zigbee2MQTT и MQTT-брокер.

Модуль нужен, чтобы:

- подписываться на топики Zigbee2MQTT;
- автоматически находить и обновлять устройства;
- держать актуальные значения свойств в runtime-кэше с живым обновлением UI;
- связывать Zigbee-свойства со свойствами и методами объектов osysHome;
- отправлять команды из osysHome обратно в Zigbee-устройства.

> [!IMPORTANT]
> Интеграция двусторонняя: Zigbee2MQTT -> osysHome и osysHome -> Zigbee2MQTT.

---

## Что получает пользователь

| Возможность | Что делает |
| --- | --- |
| MQTT-мост | Подключается к заданному брокеру и топикам |
| Автообнаружение | Создает записи устройств по входящему трафику Zigbee2MQTT |
| Редактор устройства | Позволяет настроить связи и обработку свойств |
| Живая панель | Показывает MQTT-статус, статус воркера, online/offline, батарею |
| Отправка значений | Публикует `/set` payload в топики устройства |
| Интеграция с поиском | Находит устройства и связанные свойства через общий поиск |

## Обзор интерфейса

Страница администратора:

```text
/admin/z2m
```

Верхняя панель содержит кнопку настроек, обновление списка, строку фильтра (`Фильтр по названию, описанию, модели...`) и фильтры состояния устройств.

В блоке карточек отображаются состояния `MQTT` и `Воркер`, а также сводные показатели по устройствам. Карточки можно свернуть/развернуть через `Показать карточки` и `Скрыть карточки`.

В таблице устройств показываются описание, модель, тип питания, статус доступности, связанные данные и время последнего обновления.

В редакторе устройства доступны параметры `Полный путь`, `Конвертер`, `Округление`, `Минимальный период (мс)`, режим `Обновление значений` и действие `Установить значение`.

### Разделы главной страницы

| Раздел | Описание |
| --- | --- |
| Панель инструментов | Заголовок модуля, поиск, фильтры, доступ к настройкам |
| Карточки статусов | Состояние MQTT/воркера и краткая статистика |
| Таблица устройств | Метаданные устройства, доступность, связанные значения и действия |
| Редактор устройства | Настройка связей, конвертации, периодичности и отправки значения |

> [!TIP]
> В поле топиков можно указывать несколько подписок через запятую.

## Чек-лист быстрого запуска

- [ ] Убедитесь, что Zigbee2MQTT работает и публикует данные в MQTT-брокер.
- [ ] Откройте `/admin/z2m`.
- [ ] Откройте настройки модуля.
- [ ] Заполните поля хоста, порта и топиков.
- [ ] Сохраните настройки.
- [ ] Дождитесь статуса MQTT = «Подключено».
- [ ] Откройте устройство и свяжите нужные свойства с объектами osysHome.

## Настройки MQTT

В модальном окне настраиваются:

| Поле | Обязательно | Описание |
| --- | --- | --- |
| `Хост` | Да | Hostname или IP MQTT-брокера |
| `Порт` | Да | Порт брокера, по умолчанию `1883` |
| `Username` | Нет | Логин брокера |
| `Password` | Нет | Пароль брокера |
| `Топики` | Да | Топик(и) подписки, через запятую |

Пример:

```text
zigbee2mqtt/#
```

или:

```text
zigbee2mqtt/#,custom_z2m/#
```

## Работа с устройством

### 1. Обнаружение

Устройства создаются или обновляются автоматически по входящим MQTT-сообщениям, включая bridge announce.

### 2. Открытие редактора

Откройте редактирование устройства в таблице. Можно:

- изменить `Description`;
- посмотреть метаданные (`IEEADDR`, модель, вендор, полный путь);
- настроить связи для каждого свойства;
- выбрать конвертер и параметры обработки;
- отправить тестовое значение через `Установить значение`.

### 3. Сохранение

Кнопка сохранения обновляет связи и параметры обработки свойств устройства.

## Связи с объектами osysHome

Каждое Zigbee-свойство можно связать с:

- `Linked object` + `Linked property`
- `Linked object` + `Linked method`

Примеры:

```text
Climate.outdoor_temp
```

```text
Alarm.motionDetected
```

Когда приходит новое значение:

- связанное свойство обновляется (`update` или `set` в зависимости от режима);
- связанный метод вызывается с контекстом значения.

---

## Конвертеры и параметры обработки

### Типы конвертеров

| Конвертер | Значение |
| --- | --- |
| `0` (`Default`) | Автопреобразование типичных bool-подобных значений (`on/off`, `true/false`, `open/close`) |
| `1` (`No convert`) | Оставить значение как есть |
| `2` | `online/offline -> 1/0` |
| `3` | `ColorXY <-> Hex RGB` |
| `4` | `DateTime -> timestamp` |
| `5` | `Value <-> %` (0..254 <-> 0..100) |
| `6` | `ON/OFF -> 1/0` |
| `7` | `OPEN/CLOSE -> 1/0` |
| `8` | `LOCK/UNLOCK -> 1/0` |

### Дополнительные параметры

| Параметр | Что делает |
| --- | --- |
| `Readonly` | Не регистрирует обратную запись в объект |
| `Update value` = `Only new` | Передавать только изменения значения |
| `Update value` = `All` | Передавать даже одинаковые значения |
| `Round` | Округляет числовые значения перед обработкой |
| `Minimal period (ms)` | Ограничивает частоту обработки частых обновлений (сравнение выполняется в миллисекундах) |

## Отправка значения в устройство

В редакторе устройства используйте `Установить значение` у нужного свойства.

Модуль отправляет payload в:

```text
<device_full_path>/set
```

Если в кэше состояние доступности = `offline`, сначала отправляется запрос:

```text
<device_full_path>/get
```

после чего выполняется `/set`.

## Виджет

`z2m` добавляет виджет панели управления со статистикой:

- общее количество устройств;
- количество устройств с низким зарядом;
- количество offline-устройств.

Шаблон виджета:

```text
plugins/z2m/templates/widget_z2m.html
```

## Диагностика

### MQTT не подключается

Проверьте:

- host и port брокера;
- логин/пароль;
- поле топиков не пустое;
- доступность брокера с сервера osysHome.

### Устройства не появляются

Проверьте:

- Zigbee2MQTT действительно публикует сообщения в ваши топики;
- маска топика включает сообщения устройств;
- воркер и очередь работают (карточка статуса).

### Связи не срабатывают

Проверьте:

- ссылка указывает на существующее свойство/метод объекта;
- конвертер не ломает формат ожидаемого значения;
- настройки `Readonly`/`Update value` соответствуют сценарию.
